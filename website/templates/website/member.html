{% endif %} {% comment %} ============================ Artifact Sections ============================ Each section (Projects / Papers / Videos / Talks) renders the DESKTOP count of items (page_sizes.*, e.g. 8 projects) into a grid, then a per-section ".see-more-controls" block (two buttons) directly below that grid. How the controls work (see member-load-more.js for the details): - On phones (<=576px) CSS (.is-collapsed cap in member.css) visually limits each grid to mobile_page_sizes.* (e.g. 4 projects). The other rendered cards are in the DOM, just hidden. - "Load N more ": one click lifts the mobile cap (revealing any already-rendered-but-hidden cards) AND, if the server has more, appends the next page_sizes.* batch from website:member_artifacts. The label's N is set by JS to exactly how many cards that next click will add. - "Load all ": fetches every remaining item in one request (endpoint ?all=1). Hidden when "Load more" would already load the rest. - The whole block starts hidden and is unhidden by JS only when a section actually has more to show. TRADEOFF: with JS disabled it stays hidden, so a no-JS phone visitor sees only the mobile count and can't expand. That's accepted (this site already relies on JS for the citation popover, video-age, etc.); it is strictly more than the old page showed, and there is no per-member full-listing page to fall back to. The shared data-* live on the .see-more-controls wrapper (member id, type, totals, page/mobile sizes, and the grid id to append into). ========================================================================== {% endcomment %} {% if projects %}
